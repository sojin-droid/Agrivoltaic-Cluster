# 영농형 특구 후보 클러스터 — 순위·공동 자급 계산 방법론

> 지도 패널의 **「순위 근거 (데이터 기반)」**·**「산단 매칭 · 공동 자급 기여」** 섹션이 어떻게 계산되는지 정리한 문서.
> 데이터 흐름: `필지(팜맵+소유)` → `클러스터링` → `clusters_ranked_{scen}.json` / `clusters_{scen}.geojson` → `cluster_db/clusters.db` → 지도 HTML 렌더.
> 시나리오: **S0 현행법** / **S3 영농형 특별법** / **S_MAX 이론최대**.

## 0. 기본 상수

| 상수 | 값 | 의미 | 근거 |
|---|---|---|---|
| Power Density | **0.045 kW/㎡** | 공유형 영농형 밀도 = 차광률 22.5% × 패널효율 20% | 전남 영암 실증 역산 (1,000㎡→45kW) |
| 연간 이용시간 | **1,314 h/년** | 8,760 h × 설비이용률 15% | 한국전력거래소(EPSIS) 평균 14–15% |
| 앵커 제약 | **개인소유 ≤ 50%** | 비개인(국·공유·법인·종중 등) 앵커가 사업 주도 가능해야 함 | 사업 추진 용이도 가정 |
| 최소 규모 | **≥ 50 MW** (S3/S_MAX) | RE100 특구급 규모 하한 | 신안 영농형 사례 참고 |
| 인근 산단 반경 | **≤ 10 km** | 전력 공유(자가소비·직접PPA) 현실 반경 | 분산에너지 가정 |

## 1. 후보 단위 지표 (한 클러스터당)

| 변수 | 수식 | 단위 | 비고 |
|---|---|---|---|
| 설비용량 `mw` | `면적_㎡ × 0.045 / 1000` | MW | 면적가중 |
| 연간 발전량 `gen` | `mw × 1,314 / 1000` | GWh/년 | HTML `genGwh(mw)=mw*1314/1000` |
| 개인소유 비율 `ratio` | `개인필지_면적합 / 전체_면적합` | % | **면적가중** (필지 수 아님). 앵커 제약 ≤50% |
| 최근접 산단 거리 `dist_km` | `클러스터 중심 ↔ 최근접 산단 직선거리` | km | proximity 지표 |
| 최근접 산단 수요 `demand_gwh` | 최근접 산단 1곳의 연간 전력수요 | GWh/년 | demand 지표 |

## 2. 순위 산정 — 가중치 임의성을 제거한 2단계

세 평가 지표 → **규모(scale, MW↑) · 산단근접(proximity, 거리↓) · 산단수요(demand, GWh↑)**.
"규모를 몇 % 중요하게 볼지" 같은 **가중치를 분석자가 임의로 정하지 않기 위해** 두 가지 객관 기법을 교집합으로 사용한다.

### 2-1. Pareto 비지배 (pareto_layer)
- 후보 A가 후보 B에게 **세 지표 모두에서 같거나 뒤지고, 적어도 하나는 분명히 뒤지면** → A는 B에게 "지배(dominated)"당함.
- **어떤 후보에게도 지배당하지 않는** 후보 = **비지배(non-dominated)**, `pareto_layer = 1` (최상위 프런트).
- 1층을 걷어내고 남은 것 중 비지배가 2층, … 이렇게 층을 매김.
- **UI 표기:** `pareto_layer==1` → 「**3대 기준 모두 최상위 ✓**」 / 그 외 → 「**종합 N순위군**」.
- 의미: "규모·근접·수요 어느 하나도 다른 후보에 밀리지 않는 최상위 그룹".

### 2-2. SMAA 1위 획득확률 (smaa_rank1_pct)
- **SMAA** = Stochastic Multicriteria Acceptability Analysis (확률적 다기준 수용성 분석).
- 세 지표 가중치 `(w_scale, w_prox, w_dem)`를 **합=1인 모든 조합(가중치 공간 전체)** 에서 무작위로 N회 샘플링.
- 각 가중치 조합마다 정규화 점수 `score = w_scale·n(MW) + w_prox·n(거리) + w_dem·n(수요)`를 계산하고 1위를 가림.
  - 정규화 `n(·)`: 후보군 내 min–max 스케일링 (거리는 작을수록 좋으므로 역방향).
- `smaa_rank1_pct` = **그 후보가 1위가 된 가중치 조합의 비율(%)**.
- 예) **75.2%** = "세 기준의 비중을 어떻게 잡든, 100가지 경우 중 75가지에서 이 후보가 1위".
- **UI 표기:** 「**평가비중 무관 1위 75.2%**」. ≥50%면 강조(노랑).
- **최종 순위 `smaa_rank`** = `smaa_rank1_pct` 내림차순.

> **왜 두 개를 같이 쓰나:** Pareto는 "객관적으로 최상위군"인지(정성), SMAA는 "가중치를 어떻게 줘도 1위인지"(강건성)를 본다.
> 둘의 교집합이라 분석자의 주관(가중치 선택)이 순위를 좌우하지 않는다. → UI 문구: *"평가자가 어떤 항목을 더 중시하든 순위가 거의 바뀌지 않습니다."*

## 3. 산단 매칭 · 공동 자급 기여 ★

한 특구 클러스터가 **반경 10km 내 여러 산단의 전력수요를 함께** 얼마나 감당할 수 있는지.

| 변수 | 수식 | 단위 |
|---|---|---|
| 인근 산단 목록 `nearby_complexes` | 클러스터 중심 ≤ 10km 내 모든 산단 (이름·거리·수요) | — |
| 공동 전력수요 `combined_demand_gwh` | `Σ (인근 산단 각각의 demand_gwh)` | GWh/년 |
| 클러스터 발전 `gen` | `mw × 1,314 / 1000` | GWh/년 |
| **공동 자급 기여** `contrib` | **`gen / combined_demand_gwh × 100`** | % |
| 최우선 매칭 | 최근접 산단 `nearest_complex` (`dist_km`) | — |

### 워크드 예시 — 보령 특구 후보 #1 (S3)
```
mw                  = 95 MW
gen                 = 95 × 1,314 / 1000        = 124.8 ≈ 125 GWh/년
인근 산단 (≤10km) 3곳:
  · 주포제2농공단지  1.15 km · 15.0 GWh
  · 주포농공단지     1.81 km · 17.3 GWh
  · 청소농공단지     2.99 km · 15.3 GWh
combined_demand     = 15.0 + 17.3 + 15.3       = 47.6 ≈ 48 GWh/년
공동 자급 기여       = 124.8 / 47.6 × 100       = 262 %
최우선 매칭          = 주포제2농공단지 (1.14 km)
```
→ "이 한 특구가 인근 산단 3곳의 연간 전력수요를 2.6배 감당 가능".

> ⚠ **중복 주의:** 인접한 두 특구 후보의 10km 반경이 겹치면 **동일 농지·동일 산단이 양쪽에 중복 계산**될 수 있다.
> 동시 추진 시 `smaa_rank` 우선순위로 농지/수요를 한쪽에만 배정해 중복을 정리해야 한다. (UI 하단 경고와 동일)

## 4. 데이터·필드 매핑

| UI 표시 | geojson/DB 필드 | 출처 |

| 최종 순위 | `smaa_rank` | rank1% 내림차순 |
| 인근 산단/공동수요 | `nearby_complexes[]`, `combined_demand_gwh` | §3 |
